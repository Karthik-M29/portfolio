Satellite Swarm Communication System

2026 | Arduino IDE

A satellite swarm communication system is a network of multiple small satellites that work together to transmit data using a multi-hop communication method, where each satellite acts as both a transmitter and receiver. In this project, components like Arduino UNO, nRF24L01 RF modules, AMS1117 regulator, capacitor, LED, and an LCD (for ground station) are used to simulate real satellite behavior. The working principle is that data is sent from one node and forwarded through intermediate nodes until it reaches the ground station, with the last active node responsible for final transmission. If any node fails, the system automatically reroutes the data through another available node, ensuring fault-tolerant and continuous communication. The main function of this system is to provide reliable data transfer without interruption, similar to modern satellite networks. The unique aspect of this project is its self-recovery capability, dynamic routing, and decentralized operation, unlike traditional single-satellite systems. In the future, this system can be enhanced with long-range communication, IoT sensors, and intelligent routing algorithms, making it closer to real-world satellite swarm applications used by organizations like NASA and ESA.

Day 03 - NOT Gate

08 Apr 2026 | EDA Playground

A NOT gate (inverter) is a basic digital logic gate that produces the exact opposite output of its input, meaning when the input is 0 the output becomes 1, and when the input is 1 the output becomes 0; in EDA Playground waveform view, this behavior is clearly seen as the output signal always being the inverse of the input signal over time, so whenever the input waveform goes high the output goes low and vice versa, making it easy to visually verify that the gate is correctly inverting the signal.

Day 04 - NAND Gate

09 Apr 2026 | EDA Playground

A NAND gate (NOT AND) is a digital logic gate that gives the opposite output of an AND gate, meaning the output is 0 only when all inputs are 1, and for all other input combinations the output is 1; in EDA Playground waveform, this is seen as the output staying HIGH most of the time and going LOW only when all input signals are HIGH at the same moment.

Day 05 - NOR Gate

10 Apr 2026 | EDA Playground

A NOR gate (NOT OR) is a digital logic gate that gives the opposite output of an OR gate, meaning the output is 1 only when all inputs are 0, and if any input becomes 1 the output becomes 0; in EDA Playground waveform, the output remains LOW whenever any input is HIGH and goes HIGH only when all input signals are LOW.

Day 06 - XOR Gate

11 Apr 2026 | EDA Playground

An XOR gate (Exclusive OR) is a digital logic gate that produces an output of 1 only when the inputs are different and 0 when the inputs are the same; in EDA Playground waveform, this appears as the output going HIGH whenever the input signals differ and going LOW when both inputs are either HIGH or LOW together.

Day 07 - Half Adder (First Real Circuit)

12 Apr 2026 | EDA Playground

A Half Adder is a basic digital circuit used to add two single-bit binary inputs (A and B) and produce two outputs: Sum (S) and Carry (C), where the Sum is obtained using an XOR operation (S = A ⊕ B) and the Carry is obtained using an AND operation (C = A · B); in EDA Playground waveform, when you simulate all input combinations over time, you will observe that the Sum waveform goes HIGH only when inputs are different (like 0,1 or 1,0), while the Carry waveform goes HIGH only when both inputs are HIGH (1,1), so visually the Sum toggles based on input difference and Carry produces a pulse only when both inputs rise together, clearly showing the addition behavior.

Day 08 - Full Adder

13 Apr 2026 | EDA Playground

A Full Adder is a digital circuit used to add three single-bit inputs—A, B, and Carry-in (Cin)—and produce two outputs: Sum (S) and Carry-out (Cout), where the Sum is calculated using XOR operations (S = A ⊕ B ⊕ Cin) and the Carry-out is generated when any two or more inputs are HIGH (Cout = AB + BCin + ACin); in EDA Playground waveform, when you simulate different input combinations over time, you will observe that the Sum waveform goes HIGH when an odd number of inputs are HIGH, while the Carry-out waveform goes HIGH when at least two inputs are HIGH, so visually the Sum toggles based on odd parity and Carry-out produces pulses whenever multiple inputs overlap in HIGH state, clearly demonstrating the binary addition process.

Day 09 - Ripple Carry Adder(RCA)

14 Apr 2026 | EDA Playground

A Ripple Carry Adder is a multi-bit binary adder formed by connecting multiple full adders in series, where the carry-out of each stage becomes the carry-in of the next stage, causing the carry to “ripple” from the least significant bit (LSB) to the most significant bit (MSB); in EDA Playground waveform, when inputs change, you can observe that the sum outputs of higher bits do not change instantly, instead they update only after the carry propagates through each full adder stage, creating a visible delay in the waveform from LSB to MSB, and this step-by-step carry propagation clearly shows why it is called a ripple carry adder and also highlights its slower speed compared to other adders.

Day 10 - Carry Lookahead Adder(CLA)

15 Apr 2026 | EDA Playground

A Carry Lookahead Adder (CLA) is an advanced digital adder designed to overcome the delay of ripple carry adders by calculating carry signals in advance using generate (G) and propagate (P) concepts, where G = A·B and P = A ⊕ B, allowing all carry outputs to be determined in parallel instead of waiting for ripple propagation; in EDA Playground waveform, when inputs change, you can observe that the sum and carry outputs update almost simultaneously across all bits, without the step-by-step delay seen in ripple carry adders, so the waveform shows minimal propagation delay and faster response, clearly demonstrating the high-speed performance of the CLA design.

Day 11 - 4-bit Adder-Subtractor

16 Apr 2026 | EDA Playground

A 4-bit Adder-Subtractor is a digital circuit that can perform both addition and subtraction on two 4-bit binary numbers using a control signal (usually called Mode or M), where M = 0 performs addition and M = 1 performs subtraction by taking the 2’s complement of one input (inverting B and adding 1 through the carry-in), typically implemented using full adders and XOR gates; in EDA Playground waveform, when you toggle the mode signal, you can observe that in addition mode the sum outputs change normally based on A and B, while in subtraction mode the B inputs are inverted and an initial carry is added, resulting in different output patterns, and the waveform clearly shows how the same circuit produces both operations with outputs updating accordingly along with carry/borrow behavior.

Day 12 - Multiplexer (MUX)

17 Apr 2026 | EDA Playground

A Multiplexer (MUX) is a digital switching circuit that selects one input from multiple input lines and forwards it to a single output based on select lines, for example a 4:1 MUX uses two select lines (S0, S1) to choose one of four inputs (I0–I3); in EDA Playground waveform, when the select lines change over time, you can clearly observe that the output waveform follows only the selected input signal at that moment, switching instantly as the select combination changes, so the waveform visually shows the output “jumping” between different input signals depending on the select lines, demonstrating how the MUX routes data.

Day 13 - Demultiplexer (DEMUX)

18 Apr 2026 | EDA Playground

A Demultiplexer (DEMUX) is a digital circuit that takes a single input and distributes it to one of many output lines based on select signals, for example a 1:4 DEMUX uses two select lines to route the input to one of four outputs (Y0–Y3) while all other outputs remain 0; in EDA Playground waveform, when the select lines change over time, you can observe that the input signal appears only on the selected output line, while the other outputs stay LOW, so the waveform clearly shows the input being “directed” to different outputs as the select combination changes, demonstrating how the DEMUX works.

Day 14 - Encoder

19 Apr 2026 | EDA Playground

An Encoder is a digital circuit that converts multiple input lines into a smaller number of binary output bits, for example a 4-to-2 encoder takes four inputs (only one active at a time) and produces a 2-bit binary code corresponding to the active input; in EDA Playground waveform, when different input lines are activated sequentially over time, you can observe that the output bits change to represent the binary index of the active input, so the waveform shows the output transitioning between binary values (00, 01, 10, 11) depending on which input is HIGH, clearly demonstrating how the encoder compresses information from multiple lines into a coded output.

Day 15 - Priority Encoder

20 Apr 2026 | EDA Playground

A Priority Encoder is an advanced type of encoder where multiple inputs can be HIGH at the same time, but the circuit gives priority to the highest-order input and outputs its corresponding binary code while ignoring lower-priority inputs, often along with a valid signal; in EDA Playground waveform, when multiple inputs become HIGH simultaneously, you can observe that the output reflects only the highest-priority active input, and as higher-priority inputs toggle, the output changes accordingly regardless of lower inputs, so the waveform clearly shows priority-based selection rather than simple encoding.

Day 16 - Decoder

21 Apr 2026 | EDA Playground

A Decoder is a digital circuit that converts binary input signals into multiple output lines, where each output corresponds to a unique input combination, for example a 2-to-4 decoder takes 2 input bits and activates one of four outputs (Y0–Y3); in EDA Playground waveform, when the input values change over time, you can observe that only one output line goes HIGH at a time corresponding to the input combination, while all other outputs remain LOW, so the waveform clearly shows one-hot output behavior as the active output shifts based on the input signals.
